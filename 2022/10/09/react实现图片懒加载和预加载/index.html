react解决图片懒加载和预加载

react 懒加载 预加载
react
发布日期:   2022-10-09

图片懒加载

定义：图片懒加载是鼠标滑动到哪里，图片加载到哪里。总的来说，一般页面打开，会同时加载页面所有的图片，如果图片请求太多会造成很卡很慢的现象，因此引入懒加载的概念。

原理：将图片的地址放在 data-set 属性中，由于图片并没有在 src 中，并不会发送 http 请求。比较图片到整个页面距离 Y 和页面滑动距离 X+浏览器可视区域高度 Z，若 Y 小于 X+Z,则图片就显示出来，将图片 data-set 属性换成 src

实现方案

1.安装依赖 react-lazyload

npm install react-lazyload --save 或 yarn add react-lazyload

2.在组件中使用

import React, { Component } from 'react'
import LazyLoad from 'react-lazyload'

class MyComponent extends Component {
  render() {
    return (
      <div>
        <LazyLoad height={200}>
          <img src="https://example.com/image.jpg" alt="example" />
        </LazyLoad>
      </div>
    )
  }
}

图片预加载

定义：在很多 H5 或者 APP 中会遇到，项目中存在多张背景图以及很多大图，甚至还有根据状态动态替换背景图大图等等效果，总之可以说整个项目完全就是图片叠加呈现出来的效果吧。那么在这种情况下进入页面之前就需要将图片提前预加载完成这样的话无论跳转进入哪一个页面，图片都不会出现空白或者加载慢的情况

原理：在进入页面前通过 js 加载需要的图片到浏览器缓存，浏览器刷新前，同样的图片路径请求会被跳过，直接使用缓存中的文件。

实现方案

1.安装依赖 react-image

npm install react-image --save

2.在组件中使用

import React, { Component } from 'react'
import Image from 'react-image'

class MyComponent extends Component {
  render() {
    return (
      <div>
        <Image
          src="https://example.com/image.jpg"
          alt="example"
          loader={<div>Loading...</div>}
          // 如果需要预加载多张图片，可以使用<Image>组件的preload属性来预加载多张图片。例如：
          preload={[
            'https://example.com/image2.jpg',
            'https://example.com/image3.jpg'
          ]}
        />
      </div>
    )
  }
}

总结

本地大图片预加载，动态接口返回的数据懒加载，这样可以提高用户体验
