总结react+taro开发小程序遇到过的问题

react taro 微信小程序
react
发布日期:   2022-10-15

设计稿及尺寸单位，自适应尺寸调整和固定尺寸调整的方法

在 Taro 中尺寸单位建议使用 px、 百分比 %，Taro 默认会对所有单位进行转换。在 Taro 中书写尺寸按照 1:1 的关系来进行书写，即从设计稿上量的长度 100px，那么尺寸书写就是 100px，当转成微信小程序的时候，尺寸将默认转换为 100rpx，当转成 H5 时将默认转换为以 rem 为单位的值。

如果希望部分 px 单位不被转换成 rpx 或者 rem ，最简单的做法就是在 px 单位中增加一个大写字母，例如 Px 或者 PX 这样，则会被转换插件忽略

使用 React Ref 获取不到尺寸宽高等信息

使用 React Ref 获取到的是 Taro 的虚拟 DOM，和浏览器的 DOM 相似，可以操作它的 style，调用它的 API 等。

但是 Taro 的虚拟 DOM 运行在小程序的逻辑层，并不是真实的小程序渲染层节点，它没有尺寸宽高等信息。

解决方案：使用 Taro 的 createSelectorQuery

import React from 'react'
import { View } from '@tarojs/components'
import Taro from '@tarojs/taro'

export default class Test extends React.Component {
  onReady() {
    // onReady 触发后才能获取小程序渲染层的节点
    Taro.createSelectorQuery()
      .select('#only')
      .boundingClientRect()
      .exec((res) => console.log(res))
  }

  render() {
    return <View id="only" />
  }
}

渲染标签的两种方式

Taro 通过 Vue 的 v-html 或 React/Nerv 的 dangerouslySetInnerHTML 直接渲染 HTML:

方法一：dangerouslySetInnerHTML

function helloWorld() {
  const html = `<h1 style="color: red">Wallace is way taller than other reporters.</h1>`

  return <View dangerouslySetInnerHTML={{ __html: html }}></View>
}

方法二：使用 RichText

import { RichTexta } from '@tarojs/components'

const html = `<h1 style="color: red">Wallace is way taller than other reporters.</h1>`
<RichText nodes={html} />

通过 Taro.navigateTo 传参跳转，如何获取参数

解决方案：使用 useRouter

// 传递参数
Taro.navigateTo({url:'page/b?id=123'})
import Taro, {useRouter} from '@tarojs/taro'

const Index=()=>{
  // 接收参数
  const router = useRouter();
  return(
    <View>{router?.id}</View>
  )
}
export default Index;

在分享函数 useShareAppMessage 中如何延长 1 秒后再弹出分享页

解决方案：在 useShareAppMessage 返回 Promise

useShareAppMessage(()=>{
    return new Promise((resolve)=>{
      setTimeout(()=>{
        resolve({
          title:'自定义标题',
          path:'/page/A',
          imageUrl:''
        })
      },1000)
    })
})

如何写多张图片下载完后成功/失败的回调函数

const urls = ['https://example/1.png','https://example/2.png'];

const promises = urls.map((url)=>{
  return new Promise((resolve,reject)=>{
    Taro.downloadFile({
      url,
      sucess:(res)=>resolve(res),
      fail:(err)=>reject(err)
    })
  })
})

Promise.all(promises).then((resolve)=>{
  // 成功
}).catch((err)=>{
  // 失败
})

在使用 ScrollView 组件时，iOS 出现整屏滑动的问题

微信小程序的页面是不需要左右滑动的，甚至说是不允许左右滑动的。事实上，安卓手机在默认情况下就是左右不滑动的，但苹果 IOS 手机默认是左右可滑动的，其解决方法如下：
在具体页面的顶级 view 元素设置 class=”.page”，其 CSS 样式如下：

.page {
  width: 100%;
  overflow-x: hidden;
  overflow-y: auto;
  -webkit-overflow-scrolling: touch;
}

使用 ScrollView 组件时 scrollTop 设置 0 没有置顶

源码里设置的 scrollTop 值不变时，即使设置了 0 也不起作用

解决方法：每次置顶设置一个 0~1 的随机数，实现虚拟置顶

scrollTop=Math.random();

解决 IOS 下嵌套 ifame 中的网页滚动失效亲测有效

解决方法如下：

  1. ifame 父级设置=>overflow:auto 或者 overflow-y: scroll;-webkit-overflow-scrolling: touch
  2. 设置 ifame 属性为
<iframe scrolling='yes'></iframe>

小程序使用 webview 嵌套 H5 中,H5 如何给小程序传参

方法一：在 H5 页面中传递参数给小程序，可以使用小程序提供的 JS-SDK

引入cdn: <script src="https://res.wx.qq.com/open/js/jweixin-1.6.0.js"></script>
或者 npm install weixin-js-sdk --save
import wx from 'weixin-js-sdk';

// 传递参数给小程序
wx.chooseInvoiceTitle({
  success: function (res) {
    // 获取小程序返回的参数
    console.log(res);
  },
  fail: function (err) {
    console.log(err);
  },
  complete: function () {
    console.log('complete');
  },
  // 传递的参数
  data: {
    param1: 'xxx',
    param2: 'xxx'
  }
});

在小程序中，使用 Taro.onChooseInvoiceTitle 方法监听小程序接收到的参数

componentDidMount() {
  Taro.onChooseInvoiceTitle(function (res) {
    console.log(res);
  });
}

方法二：通过 WebView 组件的 onMessage 事件监听 H5 页面发送的消息

h5 通过 postMessage 发送消息

Taro.postMessage({ data: params });

在小程序中，onMessage 监听小程序接收到的参数

// index.jsx
import Taro, { Component } from '@tarojs/taro';
import { WebView } from '@tarojs/components';

const Index=()=>{
  const handleMessage = (event) => {
    const { data } = event.detail;
    const params = JSON.parse(data);
  }

  return (
    <WebView src="https://example.com/h5-page" onMessage={handleMessage} />
  );
}

export default Index;

部分旧版存在这类问题，解决方案：加上 parent

parent.wx.miniProgram.navigateTo({
  ...
})

// 标题栏文字
Taro.setNavigationBarTitle({title:'xx'})

// 标题栏背景色

picker 省市区位置选择器

官方文档里没有详细描述，微信小程序提供了省市区选择器 picker，只需要设置 picker 的 mode 属性为’region’，省市区的内容就会自动填充。

<picker mode="region" bindchange="bindRegionChange"  value={region}>
    <view class="picker">
      当前选择：{{region[0]}}，{{region[1]}}，{{region[2]}}
    </view>
</picker>
