二、vue 文件结构

数据和方法

el 类型: string 或 HTMLElement 决定 Vue 实例会管理哪一个 DOM

三、指令

插值指令

Mustache：插值表达式

数据绑定最常见的形式就是使用 “Mustache” 语法（双大括号）的文本插值。例如：
<span>Message: {{ msg }}</span>

Mustache 标签将会被替代为对应数据对象上 msg 属性（msg定义在data对象中）的值。
无论何时，绑定的数据对象上msg属性发生了改变，插值处的内容都会自动更新,
其底层是通过数据劫持和发布订阅者模式来实现，同时利用Object.defineProperty()劫持属性的setter和getter,数据发生改变时，触发对应的监听回调渲染视图，这也就是vue的响应原理。

v-once

该指令后面不需要跟任何表达 该指令表示元素只渲染一次，不会随着数据的改变而改变，一般用于静态内容展示。

<div v-once>{{ content }}</div>

const vm = new Vue({
  el: '#app',
  data: {
    content: 'this is init data'
  }
})
vm.content = 'update data'

v-text

v-text 作用和 Mustache 比较相似：都是用于将数据显示在界面中, 不同的是 v-text 是写在属性中, v-text 通常情况下，接收一个 string 类型

v-html

解析数据中的 html 代码，渲染到页面中

v-pre

原文输出 , 不会参与编译 , 输入什么内容就展示什么内容,和 v-html 的区别在于，v-html 会解析标签

v-cloak

在某些情况下，我们浏览器可能会直接显然出未编译的 Mustache 标签。 v-cloak 指令和 CSS 规则一起用的时候，能够解决插值表达式闪烁的问题

/* css */
[v-cloak] { display: none; }

<div v-cloak>{{ root }}</div>

四、computed 属性和 watch 属性

computed 是计算属性，也就是计算值，它更多用于计算值的场景；

computed 具有缓存性，computed 的值在 getter 执行后是会缓存的，只有在它依赖的属性值改变之后，下一次获取 computed 的值时重新调用对应的 getter 来计算。

当一个属性受多个属性影响的时候就需要用到 computed,最典型的例子： 购物车商品结算的时候

<style>
table tr td{
    border:1px solid gray;
    padding:10px;

}

    border-collapse:collapse;
    width:800px;
    table-layout:fixed;
}
tr.firstLine{
    background-color: lightGray;
}
</style>

<div id="div1">

    <table align="center" >
        <tr class="firstLine">
            <td>人民币</td>
            <td>美元</td>
        </tr>
        <tr>
            <td align="center" colspan="2">
            汇率： <input type="number" v-model.number="exchange" />
            </td>
        </tr>

        <tr>

            <td align="center">
                ￥: <input type="number" v-model.number = "rmb"  />
            </td>
            <td align="center">
                $: {{ dollar }}
                $: {{ dol }}

<script>
new Vue({
      el: '#div1',
      data: {
          exchange:6.4,
          rmb:0
      },
      computed:{
         // 正常写法
          dollar: {
            get() {
              // 当 rmb和exchange变化时调用
              return this.rmb / this.exchange;
            },
            set(newValue){
                // 当dollar被修改时调用
                console.log('dollar被修改为：'+newValue);
            }
          }

          // 简写：仅调用get方法，不考虑set方法
          dol() {
              return this.rmb / this.exchange;
          }
      }
    })
</script>

watch 更多的是[观察]的作用，类似于某些数据的监听回调，用于观察 props $emit 或者本组件的值，当数据变化时来执行回调进行后续操作；

watch 无缓存性，页面重新渲染时值不变化也会执行

当一条数据影响多条数据的时候就需要用 watch,经典例子：搜索数据

<style>
table tr td{
    border:1px solid gray;
    padding:10px;

}
table{
    border-collapse:collapse;
    width:800px;
    table-layout:fixed;
}
tr.firstLine{
    background-color: lightGray;
}
</style>

<div id="div1">
    <table align="center" >
        <tr class="firstLine">
            <td>人民币</td>
            <td>美元</td>
        </tr>
        <tr>
            <td align="center" colspan="2">
            汇率： <input type="number" v-model.number="exchange" />
            </td>
        </tr>

        <tr>

            <td align="center">
                ￥: <input type="number" v-model.number = "rmb"  />
            </td>
            <td align="center">
                $: <input type="number" v-model.number = "dollar"   />
            </td>
        </tr>
    </table>

<script>

new Vue({
      el: '#div1',
      data: {
          exchange:6.4,
          rmb:0,
          dollar:0,
          clear:{
            isClear:false
          }
      },
      watch:{
         // 正常写法
          rmb:{
            handler(newValue,oldValue){
                // 数据变化时调用
                this.rmb = newValue;
                this.dollar = this.newValue / this.exchange;
            }
          },

          // 简写：仅仅监听基本类型的变化，string,number,boolean
          dollar(newValue,oldValue) {
              this.dollar = newValue;
              this.rmb = this.newValue * this.exchange;
          },

          // 深度监听，用于监听对象改变，只能用正常形式
          clear:{
            immediate:true, // 初始化让handler调用下
            deep:true, // 如果不是对象，用上述简写形式即可
            handler(newValue,oldValue){
                // 数据变化时调用
                if(newValue===true)
                this.exchange=0
            }
          }
      }
    })
</script>

五、filter 过滤器

过滤器默认是以 | 前面的的内容作为过滤器的第一个参数接收 , 自身若有参数，则作为第二个参数接收。

注意：使用过滤器不会改变原数据

例如：定义两个过滤器，分别是首字母大写和尾字母大写，然后通过如下方式实现

<style>
table tr td{
    border:1px solid gray;
    padding:10px;

}
table{
    border-collapse:collapse;
    width:800px;
    table-layout:fixed;
}
tr.firstLine{
    background-color: lightGray;
}
</style>

<div id="div1">

    <table align="center" >
        <tr class="firstLine">
            <td>输入数据</td>
            <td>过滤后的结果</td>
        </tr>
        <tr>
            <td align="center">
                <input v-model= "data"  />
            </td>
            <td align="center">
                {{ data|capitalize|capitalizeLastLetter('参数') }}
            </td>
        </tr>
    </table>

</div>

<script>

new Vue({
      el: '#div1',
      data: {
          data:''
      },
      filters:{
          // 原data不会发生变化
          capitalize(value) {
                if (!value) return '' // 如果为空，则返回空字符串
                value = value.toString()
                return value.charAt(0).toUpperCase() + value.substring(1)
          },
          capitalizeLastLetter(value,param) {
                console.log(param); // 过滤器传参
                if (!value) return '' // 如果为空，则返回空字符串
                value = value.toString()
                return value.substring(0,value.length-1)+ value.charAt(value.length-1).toUpperCase()
          }
      }
    })

</script>

七、自定义指令

作用：主要用于处理低层级的 dom 操作

指令钩子函数：（注意：在自定义指令中，this 指向 window）

bind():指令与元素绑定成功后触发（一上来）

inserted():指令所在元素被插入页面时触发

update():指令所在模板被重新解析时触发

componentUpdate :被绑定的元素所在模板完成一次更新更新周期的时候调用

unbind: 只调用一次，指令与元素解绑的时候调用

写法：分为全局指令和局部指令

  • 全局指令，Vue.directive 注册全局指令，如：自定义指令获取焦点
// 使用自定义指令
<input type="text" v-focus>

Vue.directive('focus', {
    inserted(el) {
        el.focus()// 获取元素的焦点
    }
})

带参数的自定义指令

// 使用带参数自定义指令
<input type="text" v-color='msg'>

Vue.directive('color', {
    // bind声明周期, 只调用一次，指令第一次绑定到元素时调用。在这里可以进行一次性的初始化设置
    // el 为当前自定义指令的DOM元素
    // binding 为自定义的函数形参   通过自定义属性传递过来的值 存在 binding.value 里面
    bind(el, binding){
        // 根据指令的参数设置背景色
        // console.log(binding.value.color)
        el.style.backgroundColor = binding.value.color;
    }
    });
  • 局部指令，directives 注册局部指令
<input type="text" v-focus/>

directives: {
    focus: {
        inserted(el) {
            el.focus();
        }
    }
}

组件传值

（父传子）在组件中，通过选项 props 来声明需要从父级接收到的数据

第一种方式（只接收）：

props:['name']

第二种方式（限制类型）：

props:{
  name:String
}

第三种方式（限制类型、限制必要性、指定默认值）：

props:{
  name:{
    type:String，//类型
    required:true，//必要性
    default：'老王' //默认值
  }
}

例如：父组件通过属性传参 :message=”message”

<template>
  <div>
    <child-component :message="message" @update-message="updateMessage"></child-component>
  </div>
</template>

<script>
import ChildComponent from './ChildComponent.vue';

export default {
  components: {
    ChildComponent
  },
  data() {
    return {
      message: 'Hello World'
    }
  },
  methods: {
    updateMessage(newMessage) {
      this.message = newMessage;
    }
  }
}
</script>

（子传父）子组件通过 props 接收参数，同时给父组件传值，通过$emit();

子组件：

<template>
  <div>
    <p>{{ message }}</p>
    <button @click="updateMessage">Update Message</button>
  </div>
</template>

<script>
export default {
  props: {
    message: String
  },
  methods: {
    updateMessage() {
      this.$emit('update-message', 'New Message');
    }
  }
}
</script>

需要注意的是，子组件接收父组件的数据，若数据类型是对象（引用数据类型）时，则需要深拷贝数据，否则子组件是可以改变父组件的数据的，这显然和单向数据流冲突

例如：

props:{
  obj:{
    type:Object;
  }
}
data(){
  return{
    obj1: cloneDeep(this.obj); // 深拷贝对象
  }
}

十二、mixin

本质其实就是一个 js 对象，将组件的公共逻辑或者配置提取出来，哪个组件需要用到时，直接将提取的这部分混入到组件内部即可。这样既可以减少代码冗余度

多级路由的配置方法

  1. 配置路由规则，使用 children 配置项
routes:[
    {
        path:'/about',
        component:About,
    },
    {
        path:'/home',
        component:Home,
        children:[                     // 通过children配置子级路由
            {
                path:'news',         // 此处一定不要带斜杠，写成 /news
                component:News
            },
            {
                path:'message',    // 此处一定不要写成 /message
                component:Message
            }
        ]
    }
]
  1. 跳转（要写完整路径）
<router-link to="/home/news">News</router-link>

路由的 query 参数

  1. 传递参数
<!-- 跳转并携带query参数，to的字符串写法 -->
<router-link :to="`/home/message/detail?id=${m.id}&title=${m.title}`">跳转</router-link>

<!-- 跳转并携带query参数，to的对象写法（推荐） -->
<router-link
    :to="{
        path:'/home/message/detail',
        query:{
           id: m.id,
       title: m.title
        }
    }"
>跳转</router-link>
  1. 接收参数
$route.query.id
$route.query.title

路由的 params 参数

  1. 配置路由，声明接收 params 参数
{
    path:'/home',
    component:Home,
    children:[
        {
            path:'news',
            component:News
        },
        {
      path:'message',
            component:Message,
            children:[
                {
                    name:'detail',
                    path:'detail/:id/:title', // 🔴使用占位符声明接收params参数
                    component:Detail
                }
            ]
        }
    ]
}
  1. 传递参数

特别注意：路由携带 params 参数时，若使用 to 的对象写法，则不能使用 path 配置项，必须使用 name 配置！

<!-- 跳转并携带params参数，to的字符串写法 -->
<router-link :to="/home/message/detail/666/你好">跳转</router-link>

<!-- 跳转并携带params参数，to的对象写法 -->
<router-link
    :to="{
        name:'detail',
        params:{
           id:666,
       title:'你好'
        }
    }"
>跳转</router-link>
  1. 接收参数
$route.params.id
$route.params.title

        name:'Detail',
        props:['id','title']
    }
</script>

编程式路由导航（不用 router-link）

作用：不借助 router-link 实现路由跳转，让路由跳转更加灵活
this.$router.push({}) 内传的对象与中的to相同
this.$router.replace({})
this.$router.forward() 前进
this.$router.back() 后退
this.$router.go(n) 可前进也可后退，n 为正数前进 n，为负数后退

this.$router.push({
    name:'detail',
  params:{
    id:xxx,
    title:xxx
  }
})

this.$router.replace({
    name:'detail',
  params:{
    id:xxx,
    title:xxx
  }
})

缓存路由组件

作用：让不展示的路由组件保持挂载，不被销毁

// 缓存一个路由组件
<keep-alive include="News"> // include中写想要缓存的组件名，不写表示全部缓存
    <router-view></router-view>
</keep-alive>

// 缓存多个路由组件
<keep-alive :include="['News','Message']">
    <router-view></router-view>
</keep-alive>

十七、路由守卫

作用：对路由进行权限控制

分类：全局守卫、独享守卫、组件内守卫

  1. 全局守卫 - 前置路由守卫和后置路由守卫

核心：在 router 配置中添加 meta 路由源信息

// src/router/index.js

{
    path:'/home',
    component:Home,
    children:[
        {
            path:'news',
            component:News,
      meta:{isAuth:false,title:'新闻'}
        },
        {
      path:'message',
            component:Message,
      meta:{isAuth:false,title:'消息'},
            children:[
                {
                    name:'detail',
                    path:'detail/:id/:title', // 🔴使用占位符声明接收params参数
                    component:Detail,
          meta:{isAuth:true,title:'消息详情'}
                }
            ]
        }
    ]
}

// 全局前置守卫：初始化时、每次路由切换前执行
router.beforeEach((to,from,next) => {
    console.log('beforeEach',to,from)
    if(to.meta.isAuth){ // 判断当前路由是否需要进行权限控制
        if(isLogin){ // 权限控制的具体规则
            next()    // 放行
        }else{
            alert('暂无权限查看')
        }
    }else{
        next()    // 放行
    }
})

// 全局后置守卫：初始化时、每次路由切换后执行
router.afterEach((to,from) => {
    console.log('afterEach',to,from)
    if(to.meta.title){
        document.title = to.meta.title //修改网页的title
    }else{
        document.title = 'vue_test'
    }
})
  1. 独享守卫 - 某个路由所独享
const router = new VueRouter({
    routes:[
        {
            name:'home',
            path:'/home',
            component:Home,
            meta:{title:'主页'},
            children:[
                {
                    name:'news',
                    path:'news',
                    component:News,
                    meta:{title:'新闻'},
                    // 🔴独享守卫，特定路由切换之后被调用
                    beforeEnter(to,from,next){
                        console.log('独享路由守卫',to,from)
                        if(isLogin){
                            next()
                        }else{
                            alert('暂无权限查看')
                        }
                    }
                },
                {
                    name:'message',
                    path:'message',
                    component:Message,
                    meta:{title:'消息'},
                    children:[
                        {
                            name:'detail',
                            path:'detail',
                            component:Detail,
                            meta:{title:'详情'},
                            props($route){
                                return {
                                    id:$route.query.id,
                                    title:$route.query.title,
                                }
                            }
                        }
                    ]
                }
            ]
        }
    ]
})
//导出路由器
export default router
  1. 组件内路由守卫 - 某个组件进入、离开前的守卫
<template>
    <h2>我是About组件的内容</h2>
</template>

<script>
    export default {
        name:'About',
        // 通过路由规则，离开该组件时被调用
        beforeRouteEnter (to, from, next) {
            console.log('About--beforeRouteEnter',to,from)
            if(isLogin){
                next()
            }else{
                alert('无权限查看！')
            }
        },
        // 通过路由规则，离开该组件时被调用
        beforeRouteLeave (to, from, next) {
            console.log('About--beforeRouteLeave',to,from)
            next()
        }
    }
</script>
