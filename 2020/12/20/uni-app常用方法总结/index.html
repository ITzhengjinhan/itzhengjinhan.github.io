uni-app常用方法总结以及开发中遇到的问题汇总

vue 微信小程序 uni-app
vue
发布日期:   2020-12-20

开发要用响应式样式单位 rpx

uni-app 支持的通用 css 单位包括 px、rpx

rpx 即响应式 px，一种根据屏幕宽度自适应的动态单位。以 750 宽的屏幕为基准，750rpx 恰好为屏幕宽度。屏幕变宽，rpx 实际显示效果会等比放大

uni-app 中的跳转

路由的跳转方式，同样参照了小程序 有navigator 标签 和 API 两种

二者区别：

navigator 标签： 推荐使用，有助于 SEO(搜索引擎优化)。

API： 常用跳转方式 uni.navigateTo()、uni.redirectTo() 、uni.switchTab()、uni.navigateBack()

uni.navigateTo() uni.redirectTo() uni.switchTab() uni.navigateBack()
保留当前页面，跳转到应用内的某个页面 关闭当前页面，跳转到应用内的某个页面 跳转到 tabBar 页面，并关闭其他所有非 tabBar 页面 关闭当前页面，返回上一页面或多级页面

uni.navigateBack 如何传递参数到上一个页面

方法一：采用$emit，$on以及$off
1.点击返回的这个页面触发一个事件，使用uni.$emit("事件名",对象参数)
    confirm(){
        uni.$emit('CHEvent',{CHData:this.CHData})
        uni.navigateBack({
            delta:1
        })
    },
2.在上一个页面的onShow或者onload中添加监听一个事件
    onLoad() {
        uni.$off("CHEvent");//注意：绑定监听前应解除绑定，避免产生多个监听事件
        uni.$on('CHEvent',function(data){
            console.log(data)
            })
        },
方法二：直接修改上个页面的data里的数据
1. B页面
    let pages = getCurrentPages();  //获取所有页面栈实例列表
    let nowPage = pages[ pages.length - 1];  //当前页页面实例
    let prevPage = pages[ pages.length - 2 ];  //上一页页面实例
    prevPage.$vm.navCity = "全世界";   //修改上一页data里面的searchVal参数值为1211
    uni.navigateBack({  //uni.navigateTo跳转的返回，默认1为返回上一级
        delta: 1
    });
2. A页面
  data() {
        return {
            navCity: "全国"
        }
    }

获取当前页面路由及参数的方法

方法一：在onLoad生命周期里获取
onLoad(param){
    console.log(param);
}
方法二： 通过getCurrentPages获取
let routes = getCurrentPages(); // 获取当前打开过的页面路由数组
let curRoute = routes[routes.length - 1].route //获取当前页面路由
let curParam = routes[routes.length - 1].options; //获取路由参数
// 拼接参数
let param = ''
for (let key in curParam) {
    param += '&' + key + '=' + curParam[key]
}

组件引入简化小技巧

对于在 components 目录下，并符合 components/组件名称/组件名称.vue 目录结构，就可以不用引入、注册，直接在页面中使用即可

在 H5 中使用绝对定位会被头部导航栏覆盖，小程序却可正常显示

使用–window-top 变量

.nav{
    position:absolute;
    top:var(--window-top);
    left:0;
    z-index:99
}

小程序页面栈最多 10 层。也就是说使用 uni.navigateTo 最多只能跳转 9 层页面

//页面跳转
toPage(url){
  let pages=getCurrentPages()
  if(pages.length<9){
    uni.navigateTo({url})
  }else{
    uni.redirectTo({url})//根据实际情况选择路由方法
  }
}

跨域问题

本地调试会碰到跨域问题。

解决方案：

方案一：使用 HbuilderX 内置浏览器预览。内置浏览器经过处理，不存在跨域问题。

方案二：创建一个 vue.config.js 文件，并在里面配置 devServer

// vue.config.js
module.exports = {
  devServer: {
    proxy: {
      '/api': {
        target: 'https://***.***.com',
        pathRewrite: {
          '^/api': ''
        }
      }
    }
  }
};

父组件异步数据通过 props 方式传递给子组件，子组件接收不到的问题

组件化开发中经常用到父子组件的通信，父传子子传父等数据的操作，如果父组件的数据是发请求从后端获取的异步数据，那么父组件将这个数据传递给子组件的时候，因为是异步数据，就会出现父组件传递过去了，但是子组件 mounted 钩子初始情况下是接收不到的问题

如：父组件：
<template>
  <div class="Parent">
    我是父组件
    <Child :data="data"></Child>
  </div>
</template>

<script>
import Child from "./Child.vue";
export default {
  name: "Parent",
  components: {
    Child,
  },
  data() {
    return {
      data: 0,
    };
  },
  mounted() {
    this.initValue();
  },
  methods: {
    async initValue() {
      const data1 = await this.fetchData(1);// 返回结果1
      this.data = data1.result;
      console.log(data1.result);
    },
    fetchData(num) {
      return new Promise((resolve, reject) => {
        // 这里可以使用实际的异步请求方法，比如fetch或者axios
        setTimeout(() => {
          const data = { result: num };
          resolve(data);
        }, 5000);
      });
    },
  },
};
</script>

<style scoped></style>

子组件：
<template>
  <div>
    <div>{{ newValue }}</div>
  </div>
</template>
<script>
export default {
  props: {
    data: {
      type: Number,
    },
  },
  mounted() {
    this.initValue();
  },
  data() {
    return {
      newValue: 0,
    };
  },
  methods: {
    initValue() {
      this.newValue = this.data + 1;
    },
  },
};
</script>
<style></style>

我们发现子组件的 newValue 并没有变成 2

解决方案：

方案一：父组件中使用 v-if 控制子组件渲染的时机


 <Child :data="data" v-if="data"></Child>

方案二：子组件使用 watch 监听父组件传递过来的数据

watch: {
    data() {
      this.newValue = this.data + 1;
    },
  },

vue 如何获取 dom 对象的信息

ref 方法

1. 首先有一个元素
<div class="parent">
    <div class="son" ref="btn">
</div>

2. 通过this.$refs获取son的dom
this.$refs.btn.$el.style.position = ‘fixed’

3. 获取son的父元素
this.$refs.btn.$el.parentNode.style.position = ‘fixed’

4. 获取dom的属性
this.$refs.btn.$el.getBoundingClientRect()

更多文档参考：
https://developer.mozilla.org/zh-CN/docs/Web/API/Element/getBoundingClientRect

如何做路由拦截

  1. 安装插件 uni-simple-router
npm install uni-simple-router
  1. 由于 uniapp 使用的是 page.json 配置路由，因此要使用 uni-read-pages 读取 json 文件的路由
npm install uni-read-pages
  1. 配置 vue.config.js
//vue.config.js
const TransformPages = require('uni-read-pages')
const {webpack} = new TransformPages()
module.exports = {
    configureWebpack: {
        plugins: [
            new webpack.DefinePlugin({
                ROUTES: webpack.DefinePlugin.runtimeValue(() => {
                    const tfPages = new TransformPages({
                        includes: ['path', 'name', 'aliasPath']
                    });
                    return JSON.stringify(tfPages.routes)
                }, true )
            })
        ]
    }
}
  1. 新建并写入 router.js
// router.js
import {RouterMount,createRouter} from 'uni-simple-router';

const router = createRouter({
    platform: process.env.VUE_APP_PLATFORM,
    routes: [...ROUTES]
});
//全局路由前置守卫
router.beforeEach((to, from, next) => {
    next();
});
// 全局路由后置守卫
router.afterEach((to, from) => {
    console.log('跳转结束')
})

export {
    router,
    RouterMount
}
  1. 在 main.js 引入 router.js
// main.js

import Vue from 'vue'
import App from './App'
import {router,RouterMount} from './router.js'  //路径换成自己的
Vue.use(router)

App.mpType = 'app'
const app = new Vue({
    ...App
})

//v1.3.5起 H5端 你应该去除原有的app.$mount();使用路由自带的渲染方式
// #ifdef H5
    RouterMount(app,router,'#app')
// #endif

// #ifndef H5
    app.$mount(); //为了兼容小程序及app端必须这样写才有效果
// #endif
