Ant Design Vue的问题以及开发中遇到的问题

vue

发布日期:   2023-05-06

使用 Ant Design Vue 的 a-tree-select 组件出现滚动时，下拉框会跟着页面滚动

官网解释：默认渲染到 body 上，如果遇到菜单滚动定位问题，试试修改为滚动的区域，并相对其定位

<a-tree-select
  show-search
  v-model="form.names"
  :getPopupContainer="triggerNode => triggerNode.parentNode"
</a-tree-select>

在 a-tree-select 中使用了 treeCheckStrictly=true 会使得 labelInValue 强制为 [true]

此时会把 value 类型从 string 变为 {value: string, label: VNode, halfChecked(treeCheckStrictly 时有效): string[] } 的格式

即 value: ‘b2043682ffd2413fac7481304cd7042d’——>{ label: “2 号检修班组”, value: “b2043682ffd2413fac7481304cd7042d” }

日期选择框 a-date-picker 设置可选范围在今天或之前

<a-date-picker :disabled-date="beforeToday xxxx>

在methods中
beforeToday(current){
  return current>moment()
}

日期选择框 a-range-picker 限制只能按 3 个月周期选择

<template>
  <a-range-picker
    v-model="operateDate"
    :allow-clear="true"
    :show-time="false"
    valueFormat="YYYY-MM-DD"
    @calendarChange="handlePanelChange"
    :disabled-date="disabledDate"
  />
</template>

<script>
import moment from 'moment';
export default {
  data() {
    return {
      operateDate: [], // 真实选择的时间
      startValueTemp: [], // 开始时间的暂存
    };
  },
  methods: {
    handlePanelChange(value) {
      if (value[1]) {
        // 若已选择了结束时间，清空开始时间的暂存
        this.startValueTemp = [];
      } else {
        this.startValueTemp = value[0]; // 记录开始时间的暂存
      }
    },
    disabledDate(current) {
      // 以3个月周期跨度选择
      if (!this.startValueTemp.length) return false; // 未选择开始时间不限制
      const threeMonthsLater = moment(this.startValueTemp).add(3, 'month').endOf('day');
      const threeMonthsBefore = moment(this.startValueTemp).subtract(3, 'month').startOf('day');
      return current && (current > threeMonthsLater || current < threeMonthsBefore);
    },
  },
};
</script>

a-tree 组件 autoExpandedParent 无效

当 treeNode 属性为 disabled，autoExpandedParent 无效，解决方案是通过 expandedKeys 手动设置需要展开的 key

a-tree 组件 expandedKeys 设置 key 后未展开问题

检查 treeData 数据是否有携带 key，或者 replaceFields 中 key 是否绑定到其他属性上

a-anchor 常见的问题

锚点浏览器会报错；

路由会被更改，影响哈希模式的页面路由；

affix 属性不生效，没有随着页面滚动高亮变化

/** #号后面不能为纯数字，否则无法正常定位，浏览器报错*/
<a-anchor :affix="true" :target-offset="16" :get-container="getContainer" @click="handleClickAnchor">
  <a-anchor-link v-for="(item, $index) in steps" :key="$index" :href="'#' + item.id" :title="item.title" />
</a-anchor>

/** Anchor 锚点指定滚动的容器 */
getContainer() {
  // 给组件指定渲染的容器，解决锚点不会随页面滚动而移动的问题
  return document.querySelector('.form-wrapper');
}

/** Anchor 锚点形式点击锚点 */
handleClickAnchor(e, link) {
  // 阻止点击的默认事件修改路由
  e.preventDefault();
}

如何覆盖/修改 antd-vue UI 组件样式

使用 vue 的样式穿透

方法一 /deep/ 仅支持vue2，可以穿透antd-vue UI组件
/deep/.antd input{
  font-size:12px;
}

方法二 ::v-deep 仅支持vue2，是/deep/的升级版，兼容移动端的UI
.parent::v-deep .child {
  /* 样式规则 */
}

方法三  :deep() 支持Vue3，但在Vue2中不可使用
.parent :deep(.child) {
  /* 样式规则 */
}

modal 与 form 同时使用时，destroyOnClose 属性无效问题

关闭 modal 并不会清除表单的数据，即使配置了 destroyOnClose 也无效

方案一：在关闭modal事件中添加form.resetFields();

方案二：在Form中设置preserve属性为false,然后在modal中设置destroyOnClose属性为true

读取 Excel 数据

/**
 * 读取Excel数据
 * @param {*} file 文件对象
 * @param {*} fieldMapping 字段映射
 */
export const readExcelData = async (file, fieldMapping) => {
  return new Promise((resolve, reject) => {
    function mapFields(data) {
      return data.map((row) => {
        const newRow = {};
        for (const key in row) {
          if (Object.prototype.hasOwnProperty.call(fieldMapping, key)) {
            newRow[fieldMapping[key]] = row[key];
          }
        }
        return newRow;
      });
    }
    const reader = new FileReader();
    reader.onload = (e) => {
      const data = e.target.result;
      const workbook = XLSX.read(data, { type: 'binary' });
      const sheetName = workbook.SheetNames[0];
      const sheet = workbook.Sheets[sheetName];
      const json = XLSX.utils.sheet_to_json(sheet);
      if (fieldMapping) {
        const mappedData = mapFields(json);
        resolve(mappedData);
      } else {
        resolve(json);
      }
    };
    reader.onerror = (error) => {
      reject(error);
    };
    reader.readAsArrayBuffer(file);
  });
};

父子组件通信数据污染问题以及解决方案

props的类型为基本数据类型时，子组件不能改变父组件的值；

props的类型为引用数据类型时，子组件可以改变父组件的值，为不影响数据污染，可深拷贝数据

生产环境去除 console.log

vue.config.js 中配置

configureWebpack: (config) => {
  if (process.env.NODE_ENV === "production") {
    config.optimization.minimizer[0].options.terserOptions.compress.drop_console = true;
    config.optimization.minimizer[0].options.terserOptions.compress.pure_funcs = [
      "console.log",
    ];
  }
}
