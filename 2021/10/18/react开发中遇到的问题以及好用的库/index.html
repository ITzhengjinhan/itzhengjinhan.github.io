4.使用 formily 的 Form 表单，改变一个控件值的同时，另一个控件联动变化

之前使用数据联动方式来实现，遇到多个联动会出比较混乱

import { Form, FormItem } from '@formily/antd'
const [value1,setValue1]= useState(0);
const [value2,setValue2]= useState(0);
useEffect(()=>{
    setValue2(value1)
},[value1])

<Form onSubmit={submit} {...res}>
    <formItem name="name1" component={Input} value={value1} onChange={setValue1}/>
    <formItem name="name2" component={Input} value={value2} onChange={setValue2}/>
    ...
</Form>

解决方案:引入 effects，及表单的生命周期

import { createFormActions, Form, FormEffectHooks, FormItem } from '@formily/antd';
const { setFieldState } = createFormActions();
const tag=useRef();
const change=()=>{
    FormEffectHooks.onFieldValueChange$('name1').subscribe(({value})=>{
        setFieldState('name2',(state)=>{
            state.value=value;
        })
    })
}
<Form onSubmit={submit} {...res} effects={()=>{change}}>
    <formItem name="name1" component={Input} value={value1} onChange={setValue1}/>
    <formItem name="name2" component={Input} value={value2} onChange={setValue2}/>
    ...
</Form>

使用上述方法中出现的一种窘境，就是初始化加载数据时会把 value2 变成 value1，而需求是手动编辑 value1 才能联动 value2,此时需要在表单初始化前做拦截，初始化完后再 onFieldValueChange 里联动

const change=()=>{
    FormEffectHooks.onFieldInit$('name1').subscribe(({ value }) => {
      tag.current = false;
    });

    FormEffectHooks.onFieldValueChange$('name1').subscribe(({res})=>{
        if(tag.current){
            setFieldState('name2',(state)=>{
                state.value=res;
            })
        }
        tag.current=true;
    })
}

8.antd-ProTable 中 rowSelection 配置的 onSelectNone 属性无效，且单纯使用 onChange 无法实现表格跨页多选

原因：当仅使用 onChange 实现时，其两个参数都是监听当前页面数据的变化，切换分页则返回分页下的数据，导致原始选择的数据丢失；onSelectNone 在高版本存在失效，故“取消选择”无法触发；
解决方法：加入 onSelect 和 onSelectAll

import { useState } from 'react';
import ProTable from '@ant-design/pro-table';
import { mockData , mockColumns } from './mock';
const Demo = props =>{
    const [selectedRows,setSelectedRows] = useState([]);
    let isKeepChoosed = false;
    const onSelect = (record, selected) => {
        let newData: any = [];
          if (selected) {
            newData = Array.from(new Set([...selectedRows, record[rowKey]]));
            isKeepChoosed = true;
          } else {
            newData = selectedRows.filter((item: any) => item !== record[rowKey]);
            if (newData.length !== 0) {
              isKeepChoosed = true;
            }
          }
          setSelectedRows(newData);
    }
    const onSelectAll  = (selected, selectedRows, changeRows) => {
        const ids = changeRows.map((item: any) => item[rowKey]);
          let newData: any = [];
          if (selected) {
            newData = Array.from(new Set([...selectedRows, ...ids]));
            isKeepChoosed = true;
          } else {
            newData = selectedRows.filter((item: any) => !ids.includes(item));
            if (newData.length !== 0) {
              isKeepChoosed = true;
            }
          }
          setSelectedRows(newData);
    }
    const onChange = (selectedRowKeys, selectedRows) => {
        if (data.length === 0) {
            if (isKeepChoosed) {
              isKeepChoosed = false;
            } else {
              setSelectedRows([]);
            }
          }
    }
    return (
      <ProTable
        dataSource={ mockData }
        columns={ mockColumns }
        rowKey="id"
        rowSelection={{
          selectedRowKeys: selectedRows,
          onSelect,
          onSelectMultiple: onMulSelect,
          onSelectAll: onMulSelect,
          onChange,
        }}
        tableAlertRender={() => {
          return `已选择 ${selectedRows.length} 项`;
        }}
        tableAlertOptionRender={() => {
          return false;
        }}
      />
  );
}

14.react-codemirror2 代码高亮组件的 options 属性赋值两次以上，会出现样式丢失 bug

开发中，排除了样式引入和 lineNumbers 属性导致的原因。
后发现使用 antd 的步骤条，因每步都会渲染该组件，当渲染超 2 次以上，就出现样式丢失的问题。

当我把该组件放第一步，只渲染一次时，发现没什么问题，因此就想到是多次渲染导致。
所以我在步骤组件渲染时做个判断，若处于高亮组件页面步数，再渲染 options 配置，避免多次渲染即可。

15.解决 ReactHooks 中使用 loadsh 的防抖不生效

// 防抖打印，希望输入的时候，延迟 0.5s 后打印值，发现这样使用防抖不生效

import {Input} from 'antd';
import lodash from 'lodash';
// lodash下的防抖函数
const debounce = lodash.debounce;

const getSuggestion = debounce((val:string) => {
    console.log(val);
  }, 500);

// antd的一个简单搜索文本框
const { Search } = Input;
<Search  onChange={event => getSuggestion(event.target.value)}/>

因为每次 onChange 的事件，相当于重新创建了个 debounce 防抖函数。因此我 onChange 事件触发 10 次，就会有 10 个防抖函数并执行。

解决方案：利用 useCallback、useRef 或者 ahooks 里的 useDebounceFn

import {Input} from 'antd';
import lodash from 'lodash';
// lodash下的防抖函数
const debounce = lodash.debounce;

  const getSuggestion = useRef(
    debounce(
      ()=>console.log("执行防抖，0.5秒后在请求接口"),
      500,
    ),
  ).current;
  const getSuggestion = useCallback(
    debounce(
      ()=>console.log("执行防抖，0.5秒后在请求接口"),
      500,
    ),[]
  );

17.遇到同组件需要用到多次时，不妨给组件设置不同 key，视为不同组件

react 利用 key 来识别组件，它是一种身份标识标识，就像我们的身份证用来辨识一个人一样。
每个 key 对应一个组件，相同的 key react 认为是同一个组件，这样后续相同的 key 对应组件都不会被创建

          transitionName="animated"
  >
         //这里一定要加上key
         //className是animate.css中的类名，显示不同动画
          <div key="amache" className="animated fadeInLeftBig" >
               <img src="/assets/img/history/157.png" alt="" />
          </div>
 </ReactCSSTransitionGroup>
