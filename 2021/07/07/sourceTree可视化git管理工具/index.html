sourceTree可视化git管理工具

可视化git管理工具
git
发布日期:   2021-07-07

git 是一个开源的分布式版本控制工具，可以有效、高速地解决团队开发代码托管以及异步协作开发项目。
SourceTree 是一款好用的版本管理客户端软件。
本文主要通过 sourceTree 去进行代码或文件的版本管理。教程讲述：克隆，提交，和推送

一、SourceTree 下载 安装

1.下载地址

SourceTree 下载 链接：https://pan.baidu.com/s/1W5rkiXF4DKBvUMGl5Gw5Lw 密码：djmg

2.方法/步骤

2.1 提交、推送

2.1.1 在使用 SourceTree 之前必须要先安装 Git 和 sourceTree
2.1.2 注册并登陆 Github,点击 new repository,如下图

2.1.3 输入仓库名字，点击创建，创建后的仓库是空白的，如下图

2.1.4 复制仓库地址，打开 sourcetree，点击 clone,如下图


2.1.5 在目标路径下新增文件 test1.txt,模拟暂存到暂存区，提交本地仓库，推送远程仓库过程

2.1.6 将文件暂存到暂存区

2.1.7 输入提交注释内容“新增 1”，点击提交到本地仓库。

2.1.8 推送到远程仓库

2.2 拉取和获取

2.2.1 获取和拉取。获取是将远程仓库的代码更新到本地仓库，拉取是将本地仓库的代码更新到本地工作副本。打开 sourceTree,点击 获取按钮，如下图

2.2.2 点击确定，可以看到拉取图标那里多了一个蓝底白色的数字 1，分支 master 那里也有一个灰色的数字 1，并有一个向下的箭头，这表示 有一个更新需要拉取到本地工作副本。


2.2.3 一般在实际使用中是不会先点获取，再点拉取的，因为拉代码都是直接一步到位为拉到本地工作副本的。所以直接一个拉取操作就够了。

2.3 sourceTree 中创建分支和合并分支的使用方法和使用场景

2.3.1 分支。当要新增一个功能，又要保证原始功能的正常的使用，这个时候为了不影响原功能的使用，就可以建立一个分支，在分支上进行新增功能的开发，等到新增的功能测试通过后再把分支合并到主干上。
如下图，点击分支，输入新分支名字，以当前工作副本为基础生成一个新的 develop 分支。

2.3.2 点击创建分支，可以看到多了一个 develop 分支，但是这个分支只存在于本地仓库。可以看到最新的日志记录节点上只有 master ， origin/master,develop,所以这个新分支是没有推送到远程仓库的。

2.3.3 点击推送，如下图，勾选第一个，将本地 develop 分支推送到远程 develop 分支

2.3.4 合并,将当前分支切换到 master 分支，鼠标移到到 develop 分支，右键，选择合并 develop 至当前分支，也即是主分支。


2.3.5 可以看到 master 的本地仓库多了两条需要推送的记录。如图所示的红框内图标 2，说明 develop 分支新增的两个功能已合并到本地仓库的 master 分支。

2.4 解决冲突

2.4.1 提交时发生冲突图片

2.4.2 既然提示 和远程仓库版本不一致，无法推送，那只能先拉成一致了。点击拉取图标，等执行完，就会出现下图所示。有一个待拉取的图标和一个待推送的图标，并且 test1.txt 文件的图标变成了黄色的感叹号。这个时候文件的冲突就造成了。

2.4.3 冲突文件会把两者的修改以冲突的方式合在一个文件里，等待使用者去处理。那么基本的处理思想是：要么使用其中某一个人的，要么使用两者的组合。
冲突的解决需要一个外部代码对比工具，这里使用的是 Beyond Compare 4。


2.4.4 等待一会，Beyond Compare 4 就会被打开。第一块是本地修改的版本，第二块是冲突两者（本地仓库和远程仓库）相同的版本基准，第三块是 远程仓库的修改。第四块是冲突处理后的代码

2.4.5 解决后切到 sourceTree 界面，可以看到多了一个解决冲突的中间文件，由于冲突已解决，直接右键移除，

2.4.6 可以看到冲突已解决，代码已正确合并。推送该合并。

2.5 回滚提交

2.5.1 撤销回滚。在团队协作开发中，有时候提交完代码后，发现提交错了，这个时候，撤销回滚就起了重要作用。
在最新的提交记录上，右键-》回滚提交


2.5.2 可以看到提交历史记录多了一个 Revert 反向提交

2.5.3 这个时候，反向提交只提到了本地仓库，如需撤销远程仓库的，还需推送到远程仓库

2.6 重置当前分支到此次提交

会留下记录

场景：当前 Git 有七次记录，我想回滚第三次提交。

  1. 找到第三版记录，右键点击 重置当前分支到此次提交—》强行合并-丢弃所有改动过的工作副本。如果想修改 3 分支的代码，可以选择（软合并-保持所有本地改动，或者混合合并-保持工作副本和索引)

三者区别：软合并会把 4，5，6，7 提交的文件回退到已暂存的文件区，混合合并则会返回到未暂存的文件区，强行合并则删除 4，5，6，7 文件

这里我们选择强行合并

  1. 找到远程仓库中对应的远端分支,（注意）一定要找最新的节点，右键点击 重置当前分支到此次提交—》软合并-保持所有本地改动

  1. 然后点击提交推送就好啦，这样 Git 也会留下足迹，代码也会回滚回去。

不留下记录(强制推送)

如果 git 历史记录太多，如：4，5，6，7 想删掉

找到第三版记录，右键点击重置当前分支到此次提交—》软合并

直接选择推送，（注意）勾选强制推送

若无法勾选，则需打开工具-选项-Git,勾选配置

2.7 贮藏

如果在 A 分支写了一些代码，要切换 B 分支，这个时候就需要暂时贮藏 A 分支的代码了

首先切换到 develop 分支，然后修改文件内容，并添加一个新文件。

点击确定后，发现新建的文件还在，说明新文件没有被贮藏，贮藏成功的只是被修改的文件。

再次切换 A 分支，取出贮藏数据

3.常见问题汇总

3.1 sourcetree 推送出现 Updates were rejected because the tag already exists in the remote.的错误

方法一：
推送的时候不要勾选“推送所有标签”；
这种方法治标不治本，正常的话都会勾选标签的

方法二：

1、执行命令 git pull –tags（获取所有的标签），这个地方可能会报错，问题就在这里了，would clobber existing tag 的意思是 会破坏现有的标签

2、执行命令 git pull –tags -f ，这个命令可以覆盖本地存在的标签冲突
